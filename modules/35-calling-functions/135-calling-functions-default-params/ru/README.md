У некоторых функций есть **необязательные параметры**. Это значит, что для них заранее задано значение по умолчанию, и при вызове такой параметр можно не указывать.

Рассмотрим метод `toFixed()`, который округляет число до заданного количества знаков после запятой и возвращает строку:

```javascript
const result = (10.25).toFixed(1); // => '10.3'
```

Мы передали ему одно значение — точность округления (1 знак после запятой). Но этот параметр необязателен. Если его не указать, используется значение по умолчанию `0` — округление до целого:

```javascript
console.log((10.25).toFixed()); // => '10'
```

Если нужна другая точность, её указывают явно:

```javascript
console.log((3.14159).toFixed(2)); // => '3.14'
console.log((3.14159).toFixed(4)); // => '3.1416'
```

```text
(10.25).toFixed(1)  →  аргумент: 1    →  '10.3'
(10.25).toFixed()   →  аргумент: (0)  →  '10'
                                └── значение по умолчанию
```

Количество необязательных параметров зависит от конкретной функции, но обязательные всегда идут перед необязательными.

## Сигнатура функции

У каждой функции есть **сигнатура** — описание её имени, параметров и порядка их использования. Сигнатура помогает понять, какие данные функция ожидает и что произойдёт, если параметры не указаны.

В документации необязательные параметры часто показывают в квадратных скобках. Например, сигнатура `toFixed`:

```text
toFixed([digits])
```

Здесь `digits` — количество знаков после запятой. Квадратные скобки означают, что параметр необязателен; если его не указать, по умолчанию будет `0`.

## Как работать с новыми функциями

Когда вы встречаете новую функцию, удобно действовать по простому паттерну:

1. Открыть документацию (например, MDN) и найти сигнатуру функции.
2. Посмотреть примеры использования.
3. Открыть консоль браузера или Node.js и попробовать вызвать функцию с разными аргументами.

Такой подход помогает быстро разобраться, какие у функции обязательные и необязательные параметры и какие результаты она возвращает.
