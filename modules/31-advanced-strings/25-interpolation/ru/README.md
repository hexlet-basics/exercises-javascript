Конкатенация строк через `+` работает, но читать такой код неудобно:

```javascript
const greeting = 'Здравствуйте';
const firstName = 'Анна';

console.log(greeting + ', ' + firstName + '!');
// => Здравствуйте, Анна!
```

Чем длиннее строка, тем сложнее уследить за кавычками и плюсами.

## Интерполяция

Вместо конкатенации можно использовать **шаблонные строки** (template literals). Они записываются в обратных кавычках `` ` ``, а переменные вставляются с помощью `${}`:

```javascript
const greeting = 'Здравствуйте';
const firstName = 'Анна';

console.log(`${greeting}, ${firstName}!`);
// => Здравствуйте, Анна!
```

Результат тот же, но код читается гораздо проще.

## Как это работает

Всё, что находится внутри `${}`, вычисляется как JavaScript-выражение, а результат вставляется в строку.

```javascript
const price = 100;
const count = 3;

console.log(`Итого: ${price * count} рублей`);
// => Итого: 300 рублей
```

Внутри `${}` можно использовать не только переменные, но и любые выражения: арифметику, вызовы функций, условия.

## Обратные кавычки — ключ

Шаблонная строка **обязательно** записывается в обратных кавычках `` ` ``. Одинарные или двойные кавычки не работают:

```javascript
console.log('${firstName}');  // выведет буквально: ${firstName}
console.log(`${firstName}`);  // выведет значение переменной
```
